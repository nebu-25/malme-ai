# 말메 AI 기능 정의서 (PRD)

> 작성일: 2026-06-22 / 기반 문서: docs/idea.md

## 1. 제품 한 줄 소개 & 핵심 가치

**말메 AI**는 "말로 메모하면 AI가 알아서 정리해주는" 음성 메모 앱입니다.

기존 음성 메모 앱들은 "녹음 → 텍스트 변환"까지만 도와줄 뿐, 그 이후 정리(요약, 분류, 검색)는 전부 사용자가 직접 해야 했습니다. 그 결과 메모가 쌓이기만 하고 다시 찾아보지 않는 "메모 무덤"이 되는 문제가 있었습니다(docs/idea.md의 핵심 Pain Point).

말메 AI의 핵심 가치는 **"녹음 그 이후"**에 있습니다. 말하기만 하면:
1. 텍스트로 자동 변환되고
2. AI가 한 줄 요약 + 태그 + 카테고리까지 자동으로 붙여주고
3. 나중에 태그/검색으로 바로 찾을 수 있게

즉, 말메 AI는 단순 메모장이 아니라 **한국어 특화 "제2의 뇌(Second Brain)"**를 지향합니다.

타겟 사용자: 스타트업 PM(정민수, 32세), 1인 변호사(박지연, 45세, 보안 중시), 대학원생/크리에이터(이하늘, 24세, 가격 민감) — 세 페르소나 모두 "말은 쉽게 하지만, 정리는 귀찮다"는 공통 Pain Point를 가지고 있습니다.

## 2. 핵심 기능 6가지

| # | 기능명 | 무엇을 하나 | 어떻게 동작하나 (쉬운 설명) | 어느 화면에서 |
|---|---|---|---|---|
| 1 | 음성 녹음 | 마이크 버튼을 누르고 말하면 텍스트로 바뀜 | 브라우저의 `MediaRecorder API`(휴대폰/PC의 마이크 소리를 녹음 파일로 만들어주는 표준 기능)로 음성을 녹음한 뒤, 이 음성 파일을 `OpenRouter API`(여러 AI 모델을 한 곳에서 빌려 쓸 수 있게 해주는 중개 서비스)를 통해 기본 모델 `google/gemini-2.5-flash`에게 보내 텍스트로 바꿔줍니다(STT, Speech-to-Text). 기본 모델이 응답하지 못하면 OpenRouter가 자동으로 폴백 모델로 재시도합니다(자세한 모델 구성은 7장 참고) | 녹음 화면 |
| 2 | AI 자동 정리 | 변환된 텍스트를 보면 AI가 "1줄 요약 + 태그 3개 + 카테고리 1개"를 자동으로 만들어줌 | 변환된 텍스트를 `OpenRouter API`를 통해 기본 모델 `google/gemini-2.5-flash`에게 다시 보내서, AI에게 "이 메모를 한 줄로 요약하고, 어울리는 태그 3개와 카테고리 1개를 붙여줘"라고 요청합니다. AI가 사람처럼 내용을 이해하고 정리해서 결과를 돌려줍니다. 기본 모델이 실패하면 폴백 모델로 자동 전환됩니다(7장 참고) | 처리중 화면 → 메모 상세 화면 |
| 3 | 메모 저장 | 원문 텍스트와 AI가 정리한 결과를 잃어버리지 않게 저장 | `Firestore`(구글이 제공하는, 메모를 안전하게 보관하는 클라우드 창고 같은 데이터베이스)에 원문 + 요약 + 태그 + 카테고리 + 작성일 + 작성자 정보를 한 묶음으로 저장합니다. 인터넷이 되는 곳이면 어디서든 다시 꺼내볼 수 있습니다 | (백그라운드 동작, 처리중 화면에서 진행) |
| 4 | 메모 목록 | 저장된 메모들을 날짜별 카드로 보고, 태그로 거르거나 검색할 수 있음 | 저장된 메모를 최신순으로 카드 형태로 보여주고, 태그를 누르면 그 태그가 붙은 메모만 걸러서 보여줍니다. 검색창에 단어를 입력하면 원문/요약 안에서 그 단어를 찾아줍니다 | 홈/메모 목록 화면, 검색 화면 |
| 5 | Google 로그인 | 구글 계정으로 로그인하면 내 메모만 보임 | `Firebase Auth`(구글이 제공하는 로그인 인증 서비스)를 이용해 구글 계정으로 1초 로그인합니다. 로그인한 사람마다 본인 메모만 보이도록 구분해줍니다(다른 사람 메모는 절대 안 보임) | 로그인 화면 |
| 6 | PWA (앱처럼 설치) | 웹사이트지만 휴대폰 홈 화면에 아이콘처럼 설치해서 앱처럼 쓸 수 있음 | `PWA`(Progressive Web App, 웹사이트를 진짜 앱처럼 만들어주는 기술)로 만들어서, "홈 화면에 추가" 버튼 한 번이면 앱스토어 설치 없이도 홈 화면 아이콘이 생기고 전체화면으로 실행됩니다 | 전체 앱 공통 (설치 유도는 홈/메모 목록 화면에서) |

## 3. 화면 목록

|---|---|---|

| 홈/메모 목록 화면 | 로그인 후 가장 먼저 보는 메인 화면. 내 메모를 한눈에 보고 새 메모를 시작하는 곳 | 날짜별 메모 카드 목록, 태그 필터 칩, 검색 아이콘, 녹음 시작(마이크) 버튼, 설정 아이콘, PWA 설치 유도 배너 |
| 녹음 화면 | 마이크로 말하는 동안의 화면 | 녹음 중 표시(파형/타이머), 녹음 시작/정지 버튼, 취소 버튼, 안내 문구("말씀해 주세요") |
| 처리중 화면 | 녹음이 끝난 뒤 AI가 텍스트 변환 + 정리하는 동안 보여주는 대기 화면 | 로딩 애니메이션, 진행 상태 문구("텍스트로 바꾸는 중...", "AI가 정리하는 중...") |
| 메모 상세 화면 | 메모 하나를 자세히 보는 화면 | 원문 전체 텍스트, AI 요약(1줄), 태그 3개, 카테고리, 작성일시, 재생/공유/삭제 버튼 |
| 검색 화면 | 키워드로 과거 메모를 찾는 화면 | 검색 입력창, 최근 검색어, 검색 결과 카드 목록 |
| 설정 화면 (선택) | 계정 관리 및 앱 환경설정 | 내 계정 정보, 로그아웃 버튼, 앱 정보/버전 |

## 4. 화면별 상세

### 4.1 로그인 화면
- **구성요소**: 말메 AI 로고/슬로건, "Google로 로그인" 버튼, 서비스 소개 한 줄("말하면 AI가 정리해드려요")
- **가능한 동작**: "Google로 로그인" 버튼 클릭 → 구글 계정 선택 팝업 → 인증 완료
- **이동 흐름**: 로그인 성공 시 → **홈/메모 목록 화면**으로 이동 (이미 로그인되어 있으면 앱 실행 시 이 화면을 건너뛰고 바로 홈으로 이동)

### 4.2 홈/메모 목록 화면
- **구성요소**: 날짜별 메모 카드(카드 안에 요약 1줄 + 태그 + 카테고리 + 시간 표시), 상단 태그 필터 칩 목록, 검색 아이콘, 화면 중앙/하단의 큰 마이크(녹음 시작) 버튼, 우측 상단 설정 아이콘
- **가능한 동작**:
  - 마이크 버튼 클릭 → 녹음 시작
  - 메모 카드 클릭 → 해당 메모 상세 보기
  - 태그 칩 클릭 → 해당 태그가 붙은 메모만 필터링되어 목록에 표시
  - 검색 아이콘 클릭 → 검색 화면 이동
  - 설정 아이콘 클릭 → 설정 화면 이동
- **이동 흐름**: 마이크 버튼 → **녹음 화면** / 메모 카드 → **메모 상세 화면** / 검색 아이콘 → **검색 화면**

### 4.3 녹음 화면
- **구성요소**: 녹음 중임을 보여주는 파형 또는 타이머, 정지(완료) 버튼, 취소(X) 버튼, 안내 문구
- **가능한 동작**:
  - 정지 버튼 클릭 → 녹음 종료 후 텍스트 변환·AI 정리 시작
  - 취소 버튼 클릭 → 녹음 폐기
- **이동 흐름**: 정지 버튼 → **처리중 화면** / 취소 버튼 → **홈/메모 목록 화면**(녹음 화면 진입 전 화면으로 복귀)

### 4.4 처리중 화면
- **구성요소**: 로딩 애니메이션, 단계별 진행 문구(① 텍스트로 바꾸는 중 → ② AI가 요약/태그/카테고리 정리하는 중 → ③ 저장 중)
- **가능한 동작**: 사용자 별도 조작 없음 (자동 진행). 실패 시 "다시 시도" 버튼 노출
- **이동 흐름**: 정리 및 저장 완료 → **메모 상세 화면**(방금 만든 메모)으로 자동 이동

### 4.5 메모 상세 화면
- **구성요소**: AI 요약(1줄, 강조 표시), 태그 3개(칩 형태), 카테고리 라벨, 원문 전체 텍스트, 작성 날짜/시간, 음성 재생 버튼(선택), 공유/삭제 버튼
- **가능한 동작**:
  - 뒤로가기 → 이전 화면(보통 홈)으로 복귀
  - 태그 클릭 → 같은 태그의 다른 메모 목록 보기(홈 화면에 필터 적용된 채로 이동)
  - 삭제 버튼 클릭 → 삭제 확인 후 목록에서 제거
- **이동 흐름**: 태그 클릭 → **홈/메모 목록 화면**(필터 적용) / 뒤로가기 → **홈/메모 목록 화면** 또는 **검색 화면**(어디서 들어왔는지에 따라)

### 4.6 검색 화면
- **구성요소**: 검색 입력창, 최근 검색어 목록, 검색 결과 카드 목록(메모 목록 화면과 동일한 카드 형태)
- **가능한 동작**: 키워드 입력 → 원문/요약 텍스트 내 일치하는 메모를 실시간 또는 입력완료 시 검색 / 검색 결과 카드 클릭 → 상세 보기
- **이동 흐름**: 검색 결과 카드 클릭 → **메모 상세 화면** / 뒤로가기 → **홈/메모 목록 화면**

### 4.7 설정 화면 (선택)
- **구성요소**: 로그인된 계정 이메일/프로필 사진, 로그아웃 버튼, 앱 버전 정보
- **가능한 동작**: 로그아웃 버튼 클릭 → 로그아웃 처리
- **이동 흐름**: 로그아웃 → **로그인 화면**

## 5. 전체 사용자 흐름

```
[로그인 화면]
     │ Google 로그인 버튼 클릭
     ▼
[홈/메모 목록 화면] ──── 마이크 버튼 클릭 ────▶ [녹음 화면]
     ▲                                              │ 정지 버튼 클릭
     │                                              ▼
     │                                       [처리중 화면]
     │                                  (음성→텍스트 변환 → AI 요약/태그/분류 → Firestore 저장)
     │                                              │ 처리 완료
     │                                              ▼
     │◀──────────────── 뒤로가기 ───────── [메모 상세 화면]
     │
     │ 검색 아이콘 클릭
     ▼
[검색 화면] ──── 검색 결과 카드 클릭 ────▶ [메모 상세 화면]
```

| 단계 | 사용자 행동 | 시스템 동작 | 관련 화면 |
|---|---|---|---|
| 1 | 구글 계정으로 로그인 | Firebase Auth로 인증, 내 메모 공간 입장 | 로그인 화면 → 홈 화면 |
| 2 | 마이크 버튼을 누르고 말하기 | MediaRecorder로 음성 녹음 | 녹음 화면 |
| 3 | 녹음 정지 | 녹음 파일을 OpenRouter API를 통해 기본 모델(`google/gemini-2.5-flash`)로 전송하여 텍스트 변환 (실패 시 폴백 모델로 자동 재시도) | 처리중 화면 |
| 4 | (자동) | 변환된 텍스트를 OpenRouter API의 기본 모델(`google/gemini-2.5-flash`)로 다시 보내 "요약 1줄+태그 3개+카테고리 1개" 생성 (실패 시 폴백 모델로 자동 전환) | 처리중 화면 |
| 5 | (자동) | 원문+AI 분석 결과를 Firestore에 저장 | 처리중 화면 (백그라운드) |
| 6 | 결과 확인 | 정리된 메모를 바로 상세 화면에서 확인 | 메모 상세 화면 |
| 7 | 목록에서 다시 보기 | 홈 화면에서 날짜별 카드로 확인, 태그로 필터링 | 홈/메모 목록 화면 |
| 8 | 나중에 다시 찾기 | 검색창에 키워드 입력해 과거 메모 검색 | 검색 화면 |

## 6. 데이터 저장 구조

Firestore(메모를 저장하는 클라우드 창고)에는 메모 한 건이 다음과 같이 저장됩니다. 쉽게 말해, 메모 하나당 "디지털 색인 카드" 한 장이 만들어지는 것이라고 생각하면 됩니다.

| 항목명 | 의미 (비개발자용 설명) | 예시 |
|---|---|---|
| memoId | 메모마다 부여되는 고유 번호(주민번호처럼 절대 안 겹침) | `memo_8f3a2c` |
| userId | 이 메모를 만든 사람의 구글 계정 고유 번호 (내 메모만 보이게 구분하는 용도) | `user_a1b2c3` |
| rawText | 내가 말한 내용이 그대로 텍스트로 변환된 원문 | "내일 오전 10시에 클라이언트랑 미팅하고 계약서 검토해야 함" |
| summary | AI가 만든 한 줄 요약 | "내일 오전 10시 클라이언트 미팅 및 계약서 검토" |
| tags | AI가 붙인 태그 3개 (목록 형태) | `["미팅", "계약서", "내일일정"]` |
| category | AI가 분류한 카테고리 1개 | "업무" |
| createdAt | 메모가 만들어진 날짜와 시간 | `2026-06-22 14:32` |
| audioUrl (선택) | 원본 음성 파일이 저장된 위치 (재생 기능용, 필요시) | `storage/audio/memo_8f3a2c.webm` |

> 비개발자를 위한 비유: Firestore는 "메모 카드 보관함"이고, 카드 한 장(메모 한 건)에는 항상 "누가 썼는지(userId)", "무슨 내용인지(rawText)", "한 줄로 뭔지(summary)", "어떤 분류인지(tags, category)", "언제 썼는지(createdAt)"가 같이 적혀 있습니다. 검색이나 필터링은 이 카드들 중에서 조건에 맞는 카드만 골라서 보여주는 것입니다.

## 7. AI 모델 구성 (OpenRouter)

말메 AI의 모든 AI 처리(① 음성→텍스트 변환(STT), ② 텍스트 요약/태그/카테고리 정리)는 각 AI 회사의 API에 직접 연결하지 않고, **`OpenRouter API`라는 중개 서비스 한 곳**을 통해 처리합니다. 이렇게 하면 여러 AI 모델을 같은 방식으로 호출할 수 있고, 한 모델이 잠시 멈추거나 실패해도 다른 모델로 자동으로 갈아탈 수 있습니다.

| 역할 | 모델 ID | 어디에 쓰이나 | 비고 |
|---|---|---|---|
| 기본 모델 | `google/gemini-2.5-flash` | STT + AI 정리 1차 처리 | 빠르고 합리적인 비용 |
| 폴백 1 | `google/gemini-2.5-pro` | 기본 모델 실패 시 이어서 처리 | 더 안정적인 상위 모델 |
| 폴백 2 | `openai/gpt-4o-audio-preview` | 최종 안전망, 특히 STT에 적합 | 오디오 입력을 직접 지원 |

**폴백(fallback)이란?** 기본 모델 `google/gemini-2.5-flash`로 먼저 요청을 보내고, 만약 그 모델이 응답하지 못하거나 오류가 나면 OpenRouter가 자동으로 다음 순서의 모델(`google/gemini-2.5-pro` → `openai/gpt-4o-audio-preview`)로 다시 시도합니다. 사용자는 어떤 모델이 처리했는지 신경 쓸 필요 없이 결과만 받게 됩니다.

> 비개발자를 위한 비유: 일을 처리하는 직원이 3명 대기하고 있다고 생각하면 됩니다. 평소엔 가장 빠른 1순위 직원(`google/gemini-2.5-flash`)이 일을 맡고, 그 직원이 자리를 비우면 2순위 직원(`google/gemini-2.5-pro`)이, 그마저 안 되면 비상 대기 직원(`openai/gpt-4o-audio-preview`, 특히 음성 듣기에 강함)이 대신 처리합니다. 손님(사용자) 입장에선 항상 누군가가 일을 끝내주므로 멈추는 일이 없습니다.
