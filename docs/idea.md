## 1. 잠재 타겟 고객 (페르소나)

### 페르소나 1 — "정민수" (32세, IT 스타트업 PM)
- **직업/나이**: 스타트업 프로덕트 매니저, 32세
- **사용 맥락**: 하루 평균 4~5개의 미팅(기획 회의, 디자인 리뷰, 고객 인터뷰)에 참여하며 이동 중 차나 지하철에서 아이디어를 음성으로 메모함
- **니즈**:
  - 회의 내용을 텍스트로 받아 적는 시간을 줄이고 싶음
  - 회의록을 액션 아이템 단위로 자동 분류해 Jira/Notion에 옮기고 싶음
  - 여러 회의에서 나온 메모를 주제별(고객 피드백, 일정, 버그 등)로 자동 태깅해 검색 가능하게 하고 싶음
- **핵심 페인**: 회의 후 정리에 회의 시간만큼의 추가 시간이 소요됨

### 페르소나 2 — "박지연" (45세, 1인 변호사 사무소 대표)
- **직업/나이**: 개인 변호사, 45세
- **사용 맥락**: 상담 중 또는 상담 직후 차 안에서 사건 요지와 다음 조치를 음성으로 빠르게 남김. 운전 중에도 손을 쓰지 않고 메모해야 하는 상황이 많음
- **니즈**:
  - 법률 상담 내용을 사건별로 자동 정리, 날짜/의뢰인명 태깅
  - 민감한 개인정보(의뢰인 정보)가 포함되므로 보안과 로컬 처리에 대한 신뢰 필요
  - 받아쓰기 정확도(법률 용어 인식)가 중요
- **핵심 페인**: 일반 메모 앱은 법률 전문용어 인식률이 낮고, 정보 유출 우려로 클라우드 녹음 앱 사용을 꺼림

### 페르소나 3 — "이하늘" (24세, 대학원생 / 콘텐츠 크리에이터)
- **직업/나이**: 대학원생 겸 유튜브 부업 크리에이터, 24세
- **사용 맥락**: 논문 아이디어, 강의 내용, 콘텐츠 기획 아이디어를 떠오를 때마다 음성으로 캡처. 이동 중·운동 중에도 메모 습관이 있음
- **니즈**:
  - 두서없이 말한 내용도 AI가 핵심만 요약해 정리
  - 아이디어/논문/콘텐츠 카테고리별 자동 분류
  - 무료 또는 저가 요금제 선호 (가격 민감도 높음)
- **핵심 페인**: 기존 메모 앱은 녹음만 해주고 정리는 본인이 직접 해야 해서 메모가 쌓이기만 하고 활용되지 않음

## 2. Pain Point 분석

| 구분 | 현재 문제점 |
|---|---|
| **메모 습관 단절** | 손으로 타이핑하기 번거로운 상황(운전, 이동, 회의 중)에서 음성 메모는 편하지만, 이후 "정리"가 안 되어 방치됨 |
| **정리의 누적 부채** | 음성 메모/녹음 파일이 쌓일수록 다시 듣고 정리하는 데 드는 시간이 기하급수적으로 늘어남 ("녹음은 쉬운데 정리는 지옥") |
| **검색/활용 불가** | 텍스트화되지 않은 음성 파일은 키워드 검색이 불가능해 필요할 때 못 찾음 |
| **맥락 손실** | 며칠 지나면 "이 메모를 왜 남겼는지" 맥락이 사라져 재활용도가 낮음 |
| **분류 노동** | 업무/개인/아이디어/할일 등을 사용자가 직접 폴더링·태깅해야 하는 번거로움 |
| **낮은 한국어 인식 정확도** | 사투리, 전문용어(법률·의료·IT), 잡담체 구어 인식에서 글로벌 STT 서비스의 한국어 정확도가 떨어지는 경우가 많음 |
| **개인정보 우려** | 민감한 음성 데이터가 해외 서버로 전송·저장되는 것에 대한 불안감 (특히 변호사, 의료진, 상담사 등) |
| **단순 받아쓰기의 한계** | 기존 STT 앱은 "텍스트 변환"까지만 하고 구조화/요약/분류는 사용자가 직접 해야 함 |

## 3. 경쟁 서비스 분석

| 서비스 | 핵심 기능 | 장점 | 단점 | 가격 (최신 확인 필요) |
|---|---|---|---|---|

| **Otter.ai** | 영어 중심 STT, 실시간 자막, 요약 | 글로벌 표준급 영어 인식률, Zoom/Google Meet 연동 강력 | 한국어 인식 정확도 낮음, UI/UX가 한국 사용자에게 익숙하지 않음 (최신 확인 필요) | Freemium + 유료 플랜 (최신 확인 필요) |
| **Notion AI** | 문서 내 AI 요약/정리/Q&A | 강력한 정리·구조화 생태계, 기존 노트와 통합 용이 | 음성 입력 자체가 핵심 기능이 아님(STT는 보조적), 실시간 음성 캡처 경험 약함 (최신 확인 필요) | Notion 요금제에 부가 (최신 확인 필요) |
| **삼성 음성 녹음(Voice Recorder)** | 기기 내 녹음, 기본 STT(빅스비 연동) | 갤럭시 기본 탑재로 접근성 높음, 오프라인 녹음 안정적 | AI 요약/분류 기능 부재 또는 초기 단계, 안드로이드 갤럭시 생태계 한정 (최신 확인 필요) | 무료 (기기 내장) |
| **Apple 음성 메모** | 기기 내 녹음, iOS 받아쓰기 | iPhone 기본 탑재, 디바이스 간 동기화(iCloud) | AI 정리/요약 기능 거의 없음(최근 일부 추가, 최신 확인 필요), iOS 생태계 종속 | 무료 (기기 내장) |
| **Whisper 기반 서드파티 앱들** (예: Wispr Flow 등) | OpenAI Whisper 기반 STT, 텍스트 변환 | 오픈소스 기반 커스터마이징 가능, 다국어 지원, 빠른 발전 속도 | 앱별로 품질 편차 큼, 한국어 후처리(맥락 요약/분류) 품질이 검증 안 된 경우 많음 (최신 확인 필요) | 앱마다 상이 (최신 확인 필요) |

**시장 공백(Gap) 요약**:
대부분의 경쟁 서비스는 ① "회의록" 중심(클로바노트, 다글로, Otter)이거나 ② "단순 녹음/받아쓰기"(삼성, 애플) 중 하나에 머물러 있음. **일상적인 개인 메모를 자동으로 구조화·요약·분류**해주는, 즉 "회의가 아닌 순간의 생각"까지 포괄하는 올인원 한국어 특화 서비스는 상대적으로 비어 있는 영역.

## 4. 추천 핵심 기능

### MVP 필수 기능
- 고정밀 한국어 STT (구어체, 사투리, 침묵/끊김 보정)
- 원탭 녹음 시작/종료 (위젯, 잠금화면, 워치 연동 등 빠른 접근성)
- 자동 요약 (긴 음성도 핵심 문장으로 압축)
- 자동 카테고리 분류/태깅 (업무, 아이디어, 할일, 개인 등)
- 텍스트 검색 기능 (키워드로 과거 메모 탐색)
- 기본적인 액션 아이템 추출 ("~해야 한다" 류 문장 자동 인식)
- 메모 편집/수정 UI (AI 결과 보정 가능)
- 데이터 암호화 및 기본 개인정보 보호 정책

### 추후(확장) 기능
- 화자 분리 (다자 회의 대응)
- 캘린더/Notion/Slack/Jira 등 외부 툴 연동(자동 동기화)
- 음성 톤/감정 분석 (예: 회고 메모의 감정 흐름 파악)
- 반복 패턴 학습 (사용자별 자주 쓰는 분류 체계 자동 학습)
- 멀티 디바이스 실시간 동기화 (워치 → 폰 → PC)
- 팀/조직용 공유 워크스페이스 (B2B 확장)
- 법률·의료 등 도메인 특화 용어 사전 추가 옵션
- 오프라인 모드(온디바이스 STT) — 보안 민감 사용자 대상
- 음성 명령으로 메모 검색/실행 ("지난주 회의 내용 요약해줘")

## 5. 성공 고려사항

### 기술적 고려사항
- **음성인식 정확도**: 한국어 구어체, 전문용어, 잡음 환경에서의 정확도가 핵심 경쟁력. 자체 파인튜닝 또는 STT 벤더 선택(클로바, Whisper, Google STT 등) 전략 필요
- **AI 비용 구조**: STT + LLM 요약/분류를 매 메모마다 호출하면 사용량 증가 시 비용 부담이 커짐. 캐싱, 배치 처리, 모델 경량화 전략 필요
- **지연시간(latency)**: 음성 종료 후 결과가 즉시(수 초 내) 보여야 사용자 체감 만족도가 높음. 비동기 처리와 progressive UI 설계 중요
- **개인정보/보안**: 민감 정보(법률, 의료, 회사 기밀) 포함 가능성이 높아 종단간 암호화, 온디바이스 처리 옵션, 국내 서버 운영 등이 신뢰 확보에 중요
- **오프라인 대응**: 네트워크 불안정 환경(지하철, 산간 지역)에서도 기본 녹음은 가능해야 함

### 사업적 고려사항
- **수익 모델**: Freemium(월 일정 시간 무료) + 구독제(무제한/고급 기능)가 표준적. B2B(팀 플랜, 회의록 특화)로 확장 시 ARPU 상승 가능
- **시장 규모**: 한국 내 생산성 앱/AI 메모 시장은 빠르게 성장 중이나, 글로벌 대비 작음. 초기엔 한국어 특화로 니치를 선점하고 이후 다국어 확장 검토
- **진입장벽**: STT 기술 자체의 진입장벽은 낮아지는 추세(오픈소스 Whisper 등)이므로, 차별화는 "정리/분류/UX 경험"에서 나와야 함
- **고객 락인(Lock-in) 전략**: 누적된 메모 데이터와 학습된 분류 체계가 쌓일수록 이탈 비용이 높아지는 구조 설계 필요
- **파트너십**: 캘린더, 협업툴(Notion, Slack) 등과의 연동이 채널 확장과 자연스러운 사용 빈도 증가에 기여

## 6. 차별화 전략

1. **"녹음 그 이후"에 집중**: 경쟁사들이 텍스트 변환(STT)에 집중하는 반면, 말메 AI는 변환 이후의 **구조화 → 요약 → 분류 → 액션화**까지의 풀 파이프라인을 핵심 가치로 포지셔닝
2. **한국어 일상 구어체 특화**: 회의록 중심의 클로바노트/다글로와 달리, 비정형적이고 두서없는 "일상 아이디어 메모"에 강한 한국어 NLU 튜닝
3. **자동 학습형 분류 체계**: 사용자의 메모 패턴을 학습해 카테고리/태그를 스스로 진화시키는 개인화 분류 엔진
4. **빠른 캡처 UX**: 위젯, 잠금화면, 워치 연동 등 "말하고 싶을 때 0.5초 안에 녹음 시작"할 수 있는 마찰 없는 진입 경험
5. **신뢰 가능한 프라이버시**: 민감 정보를 다루는 전문직(변호사, 상담사, 의료진) 타겟을 위한 온디바이스 처리/암호화 옵션을 명확한 차별점으로 마케팅
6. **올인원 라이프 메모 허브**: 회의록 전용 도구가 아닌, 업무·아이디어·할일·개인 기록을 모두 아우르는 "제2의 뇌(Second Brain)"형 포지셔닝으로 Notion/Apple Notes 사용자층까지 흡수
